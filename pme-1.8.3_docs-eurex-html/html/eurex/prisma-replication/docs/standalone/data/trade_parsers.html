Portfolio

Portfolios can be imported from data files in several supported formats.

ETD formats

ETD position files can be imported from the following formats:

  • Eurex ETD CSV
  • FIXML

OTC formats

OTC trade files can be imported from the following formats:

  • Eurex OTC CSV
  • FpML

Sensitivity-based formats

It is also possible to represent one or more OTC trades as sensitivities to risk factors, rather than providing trade definitions. These can be loaded from the following file format:

  • Eurex Sensitivities CSV

Loading programmatically

For more advanced use cases, it is also possible to build portfolios by constructing OpenGamma domain objects programmatically. This gives the user full access to functionality supported by the trade model. For further information, see Trade Model.
